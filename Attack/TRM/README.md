# TRM (Text Rendering Mode) 隐形文本攻击工具

TRM是一个用于演示PDF文本渲染模式3（Text Rendering Mode 3）隐形文本攻击的工具集，包含了基本示例和高级示例。

## 功能介绍

本项目演示了如何利用PDF规范中的文本渲染模式（Text Rendering Mode）来创建隐形文本，这些文本在视觉上不可见，但可以被文本选择工具选中和复制。

### 基本示例 (TRM_main.py)

向现有PDF文件添加基本的文本渲染模式3隐形文本攻击：

- 读取用户提供的输入PDF文件
- 使用文本渲染模式3（不可见模式）创建隐形文本
- 将隐形文本添加到PDF中
- 保存修改后的PDF文件

### 高级示例 (TRM_ELIT.py)

向现有PDF文件添加三种更高级的TRM隐形文本攻击技术：

1. **PoC #1: Tr=3 + 极小文本矩阵**
   - 使用文本渲染模式3和极小的文本矩阵缩放（~1e-5）
   - 使文本的边界框（bbox）接近零
   - 更难被检测工具发现

2. **PoC #2: Tr=3 + TJ碎片化**
   - 使用文本渲染模式3
   - 将文本分割成单个字符的TJ数组
   - 为每个字符添加极端的偏移量
   - 增加文本提取的难度

3. **PoC #3: Tr=3 + /Artifact标记内容**
   - 使用文本渲染模式3
   - 将隐形文本包裹在/Artifact标记内容中
   - 利用某些PDF处理工具对/Artifact内容的特殊处理

## 依赖环境

- Python 3.x
- reportlab 库：用于生成PDF文件和创建隐形文本
- PyPDF2 库：用于处理PDF文件和添加隐形文本

## 安装依赖

```bash
pip install reportlab PyPDF2
```

## 使用方法

### 运行基本示例

```bash
python TRM_main.py
```

按照提示输入：
- 原始PDF文件的完整路径
- 输出PDF文件的路径（可选，直接按回车使用默认路径）

运行后将生成一个添加了TRM隐形文本的PDF文件（默认输出文件名：`trm_attack_原始文件名.pdf`）。

### 运行高级示例

```bash
python TRM_ELIT.py
```

按照提示：
1. 选择攻击类型（1-3）
2. 输入原始PDF文件的完整路径
3. 输入输出PDF文件的路径（可选，直接按回车使用默认路径）

运行后将生成一个添加了相应高级TRM攻击的PDF文件，默认输出文件名格式为：
- 攻击类型1：`elit_tr3_tiny_matrix_原始文件名.pdf`
- 攻击类型2：`elit_tr3_tj_fragmented_原始文件名.pdf`
- 攻击类型3：`elit_tr3_artifact_原始文件名.pdf`

## 技术原理

### 文本渲染模式3

PDF规范定义了多种文本渲染模式，其中模式3表示"不可见（Invisible）"：
- 文本不被绘制（既不填充也不描边）
- 但文本仍保留在页面的文本流中
- 可以被文本选择工具选中和复制

### 高级技术

1. **极小文本矩阵**：通过设置极小的缩放因子，使文本的边界框接近零，降低被检测工具发现的概率。

2. **TJ碎片化**：将文本分割成单个字符，使用TJ操作符逐个绘制，并为每个字符添加随机偏移，增加文本提取和分析的难度。

3. **/Artifact标记内容**：利用PDF的标记内容功能，将隐形文本标记为/Artifact（辅助内容），某些PDF处理工具可能会忽略或特殊处理这类内容。

## 生成的PDF内容

### 基本示例PDF

- 保留原始PDF的所有内容
- 添加隐形文本："INVISIBLE_TEXT: If you can copy this, Tr=3 worked."

### 高级示例PDF

每个高级示例都：
- 保留原始PDF的所有内容
- 使用特定技术添加相应的隐形文本：
  - 攻击类型1："P1_INVISIBLE_TINY_TM: Tr=3 + Tm scale ~1e-5."
  - 攻击类型2："P2_TJ_FRAG"
  - 攻击类型3："P3_ARTIFACT_TR3: invisible text in /Artifact BMC ... EMC"


## 检测方法

要检测使用文本渲染模式3的隐形文本，可以：

1. 使用文本选择工具在可疑区域尝试选择文本
2. 使用PDF解析工具分析页面内容流，查找"3 Tr"或"3 Tr"命令
3. 使用专门的PDF安全检测工具
